# Apexカスタムリザルト集計ツール

Apex Legendsのカスタムマッチで、スクリーンショットから戦績（順位・チーム名・キル数）を読み取って、スプレッドシートに書き込むツールです。

## ⚠️ 最初に確認：解像度について
- スクリーンショットは必ず **1920×1080** の解像度で撮影してください。
- これ以外のサイズだと、文字を読み取る場所がズレて正しく動作しません。

## 🛠️ セットアップ手順

### 1. ツールのダウンロード
ターミナル（PowerShellなど）を開き、以下のコマンドを順番に実行します。

   # リポジトリをコピー
   git clone https://github.com/umakoUMUM/apex-custom-tools.git

   # フォルダの中に移動
   cd apex-custom-tools

### 2. Python環境の準備
※パソコンにPythonが入っていない場合は、公式サイト(python.org)からインストールしてください。

   # 仮想環境を作成
   python -m venv .venv

   # 仮想環境を有効化（Windows）
   .\\.venv\Scripts\activate

※もし「スクリプトの実行が無効」というエラーが出たら、以下を実行してから再度 activate してください。

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope Process

### 3. 必要なライブラリのインストール
   pip install -r requirements.txt


### 4. Google Cloud (GCP) の設定
1. **APIの有効化**: GCPコンソールでプロジェクトを作成し、以下3つを有効にします。
   - Cloud Vision API / Google Sheets API / Google Drive API
2. **キーの発行**: サービスアカウントを作成し、「JSON形式」の鍵を発行します。
   - ダウンロードしたファイルを `credentials.json` にリネームしてください。
3. **メールアドレスのコピー**: サービスアカウントのメールアドレスをメモしておきます。

### 5. スプレッドシートの準備
1. 集計用のスプレッドシートを新規作成します。
2. 右上の「共有」から、先ほどの**サービスアカウントのメールアドレス**を「編集者」として追加します。
3. ブラウザのURLから「スプレッドシートID」をコピーします。

### 6. ファイルの配置
以下のファイルをルート（main.pyと同じ場所）に用意します。
- **.env**: 以下の一行を書き込みます。
  SPREADSHEET_ID=コピーしたIDを貼り付け
- **credentials.json**: 手順4で取得した鍵を置きます。
- **config.yaml**: 参加チーム数（TOTAL_TEAMS）などが正しいか確認します。

## 📝 使い方
1. `screenshots` フォルダにリザルト画像（1920x1080）を入れます。
2. プログラムを実行します。
   python main.py

3. 解析が終わると画像は自動的に `processed` フォルダに移動します。

## 📊 セルの色について
- **オレンジ色（補正）**: ツールが自動で数値を直した箇所です（例：60→9）。
- **赤色（異常）**: 順位が0位など、読み取りミスの可能性がある箇所です。手動で直してください。

## 🔍 テスト用ツール
- **visual_check.py**: 画像の切り出し範囲が正しいか確認できます。
